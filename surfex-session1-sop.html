Common issues during setup. Find the symptom, run the fix.
npm install fails with permission error
Run with sudo:
sudo npm install
Bot starts but doesn't respond in Telegram
Two most common causes: (1) TELEGRAM_BOT_TOKEN is wrong — must match exactly what BotFather gave. (2) ALLOWED_CHAT_ID is wrong — must be the number from getUpdates, not a username or @handle. To verify the correct ID: open browser, paste https://api.telegram.org/botYOUR_TOKEN/getUpdates (replace YOUR_TOKEN), find "chat":{"id":XXXXXX — that exact number goes in .env with no quotes. Check both, restart bot.
Error: ENOENT — vault path not found
The VAULT_PATH in .env doesn't match the actual folder. Confirm exact path:
ls ~/Documents/SurfexBrain
Copy the output exactly into VAULT_PATH= in .env. No trailing slash.
node --version not found after installing Node.js
Close Warp completely and reopen it. The PATH updates on new shell sessions only.
gh auth status shows not logged in
Run the auth flow:
gh auth login
Choose: GitHub.com → HTTPS → Login with a web browser → authorize in browser.
git clone fails: repository not found
Collaborator invite not yet accepted. Check Mike's email for noreply@github.com — he needs to accept before cloning.
Error: Cannot find module on npm start
npm install didn't complete cleanly. Run it again:
cd ~/claudeclaw && npm install
Voice note arrives but no transcription
GROQ_API_KEY missing or blank in .env. Add it and restart the bot. Without Groq, text messages work but voice notes skip transcription.
